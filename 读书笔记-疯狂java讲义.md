## chapter 1：java概述与开发环境

#### 1、java既是编译型，又是解释型语言。

* 编译型语言：通过编译器，将高级语言源代码一次性“翻译”成本地机器码，生成的可执行程序脱离开发环境，在特定平台上独立运行，效率高。但是移植性较差，如果要跨平台执行，通常要把源代码在不同平台上用不同的编译器编译，操作比较麻烦。
* 解释型语言：解释器是把编译和解释的步骤合并在一起同时完成。运行效率较低，且无法脱离解释器独立运行。但跨平台容易。
* java语言是先通过编译器编译成平台无关的字节码文件，然后由JVM转换为可执行的机器码。不同平台的JVM实现了java语言的跨平台。

#### 2、JRE、JDK的含义

* **JRE**：java运行时环境（*Java Runtime Environment*），运行java程序必须的条件。包括JVM，类加载器，字节码校验器及大量的基础类库等。除了JVM，还包含运行java程序的其他环境支持。
* **JDK**：java开发工具套件（*Java SE Development Kit*）。包括编译器、JRE及常用的java类库等。

#### 3、内存管理的变更

* java 8对HotSpot JVM中的永久代被剔除，改为使用本地内存来存储类的元数据信息，并称为元空间（Metaspace）；
* java 7提供了G1垃圾回收器来替代原有的并行标记/并行垃圾回收器。

## chapter 2：理解面向对象

#### 1、OOP方法的基本特征：封装、继承、多态。

* 封装：是指将对象的实现细节隐藏起来，然后通过公共方法暴露该对象的功能；
* 继承：子类继承父类后，子类作为一种特殊的父类，将直接获得父类的属性和方法；
* 多态：是指子类对象可以直接赋值给父类变量，运行时依然表现出子类的行为特征。这种方法可以使一个对象执行同一个方法时，可以表现出多种行为特征；
* 抽象：忽略无关的状态和行为，提取主要的、相关的属性和方法；

## chapter3：数据类型与运算符

#### 1、java是强类型语言，具体表现为：

* 所有的变量使用之前要先声明；
* 变量只能接受类型与之匹配的值；

#### 2、标识符定义规则

* 数字、下划线、字母和$，但是不能以数字开头；
* 不能是java的关键字和保留字，但是可以包含关键字和保留字；
* 不能有空格；
* 特殊字符只能有$；

> 关键字：java中累计有50个关键字，如class、int、double、void等；
>
> 保留字：goto和const，保留未使用；
>
> 特殊的字面量：true、false、null；

​		除了基本类型，还有引用类型。引用类型包括类、接口和数组类型。

​		空类型是null值的类型，该类型没有名称，所以不能声明null类型的变量或者转换到null类型。null是null类型变量唯一的值；

#### 4、整数的二进制表示

​	以0b或0B开头（十六进制以0x或0X开头），计算机中以补码的形式存在。

* 原码：正数符号位为0，负数为-1；
* 反码：除符号位以外，按位取反；
* 补码：反码加1；

> 计算机中减法的借位不好实现，使用补码可以将减法转变为加法。例子如下：
>
> 56-28 => 56-28+99-99=>56-28+99+1-100=>(99-28+1)+56
>
> 括号的部分即为-28的补码形式。
>
> 总结，为了便于计算减法，将int等数值类型的取值范围分成两部分，0~2^31-1表示正数，高出的部分表示负数，所以负数的符号位为1；

#### 5、浮点类型中的无穷大、无穷小和非数

* 无穷大：表示某非0.0的浮点正数除以0.0；

* 无穷小：表示某非0.0的浮点负数除以0.0；
* 非数：0.0/0.0；

> 无穷大、无穷小、非数分别对应Double或Float类的POSITIVE_INFINITY、NEGATIVE_INFINITY和NaN。
>
> 所有的无穷大和无穷小是相等的。但非数是不相等的，甚至非数不等于非数；

#### 8、表达式类型的自动提升规则

​		当一个表达式中包含多种基本类型的值时，整个算术表达式的数据类型将发生自动提升。

* 所有byte、short和char都将被提升到int类型后，在计算。
* 整个算数表达式的数据类型自动提升到与表达式中最高等级操作数同样的类型。

#### 9、字符串字面量

​		当程序第一次使用某个字符串直接量时，java会把这个直接量缓存到JVM的常量池，如果后面还需要用到这个直接量，就直接使用常量池中的直接量。

#### 10、求余运算符

​		求余运算符需要两个操作数，返回第一个操作数除以第二个操作数后的余数。因此，操作数类型是整型或浮点型。如果第二个操作数为0或0.0，对应的结果与除法类似——整型操作数对应产生除0异常，如果是浮点型操作数，则返回非数。

#### 11、位运算符

* &：按位与。同为1，则返回1；
* |：按位或。一位为1，就会返回1；
* ^：同则为0，不同则为1；
* ~：将数的每一位按位取返；
* <<：左移
* \>\>：右移
* \>\>\>：无符号右移。因为第一位是符号位，所有只有右移。空出的位置填0。

> 移位运算的规则：
>
> * 低于int类型的操作数总是先自动类型转换为int类型后再移位。
> * 对于移位操作数大于32的情况，系统会先将移位操作数对32求余，得到的结果再作为移位的位数。
> * 同理如果是对long类型的整数移位，则移位操作数大于64时，规则与上一条一致。
> * 移位运算符返回的是一个新的数，原来的操作数不会改变。

#### 12、比较运算符

​		基本类型的变量、值不能和引用类型的变量、值使用==比较；boolean类型的变量、值不能与其他任意类型的变量、值使用==比较；两个引用类型没有继承关系，也不能用==比较；

#### 13、逻辑运算符

* &&：同为true，则返回true；
* &：同为true，则返回true，不短路；
* ||：一个为true，则返回true；
* |：一个为true，则返回true，不短路；
* ！：逻辑非，你懂的。
* ^：逻辑异或，不同则为true，同则为false；

#### 14、switch分支语句

​		switch语句后面的控制表达式的数据类型只能是byte、short、char、int和枚举类型。java7加入了String类型，不能是StringBuffer和StringBuilder。

## chapter4 流程控制与数组

#### 1、数组——一种数据类型

​		数组本身就是一种数据类型。数组中既可以存储基本类型，也可以存储引用类型的数据。理论上数组只能存储一种数据类型的数据，但是考虑到引用类型中存在类的继承关系。父类数组中可以存储不同子类的对象实例，造成了数组中可以存储不同类型的假象。数组初始化的时候，分配有效内存空间后必须赋初始值，要么是系统自动分配，或程序员指定。

> 静态初始化：初始化时由程序员显示指定每个数组元素的初始值，由系统决定数组长度。
>
> 动态初始化：初始化时程序员只制定数组长度，由系统为数组元素分配初始值。
>
> 两者不可并存。

* byte、short、int和long类型数组：默认为0；
* float、double数组：默认为0.0；
* char数组：默认是'\u0000'；
* boolean数组：默认是false；
* 引用类型(类、接口、数组)：默认是null；

#### 2、多维数组的扩张

​		java不想javascript和Ruby等弱类型的语言，不能通过给数组元素赋值另一个数组来达到多维数组扩充的目的。因为java是强类型的语言，声明的时候定义了数组元素的数据类型，就必须一致。否则，还请用Object[]来实现扩充。
